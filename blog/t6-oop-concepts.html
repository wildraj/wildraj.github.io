OOP vs Functional Programming

Posted on 6/22/15

For years there has been a debate over Object-Oriented Programming(OOP) and Functional Programming(FP). OOP is a programming paradigm based on the concepts of objects. They contain attributes (data) and behaviors (methods). For example, here is the Calculator object below:

Class Calculator
def initialize(a,b)
@a = a
@b = b
end
def add
@a + @b
end
def subtract
@a - @b
end
end

This is a Calculator class. Its attributes are the variables @a and @b since they are used in a couple mathematical operations. Its methods are add and subtract since they perform operations with @a and @b. That is OOP. Now functional programming is a little strange. Basically it is when functions are used as the building blocks of the program, nothing more. Here is a code example:

int add (int x, int y)
//A {
int z;
//B z = x + y;
return (z);
//C }
main ()
{
int i, j, k;
i = 10;
j = 20;
k = add(i, j);
//D printf (“The value of k is%d\n”, k);
//E }

"k" is the result of the function "add" that returns the sum of "x" and "y". The starting point of the whole code starts at the beginning of the "main" function. As you can see OOP and FP are similar, but perform differently. Some people who are in debate of OOP and FP prefer OOP while others like FP. Here we'll discuss some pros and cons of OOP and FP.

OOP
(Pros):
1. easy to use for other programs
2. easy to use whenever finishes
3. can hide certain data to prevent programmers from tampering with values they shouldn't

(Cons):
1. methods can be difficult to implement
2. much larger than other programs, takes up space in memories
3. which leads to the fact that the execution is slow

FP
(Pros):
1. avoids large amounts of code to define states
2. fast compared to OOP
3. prevents side effects, meaning it can't change global variables by accident

(Cons):
1. can be difficult to implement for other programs
2. most languages are not practical for real time systems
3. less efficient than other languages
